# CraftRadar — Техническое задание (полное)

## 1. Общее описание

**CraftRadar** — мониторинг серверов Minecraft. Каталог серверов с автоматическим сбором статистики, рейтингом, голосованием и аналитикой.

Владельцы серверов регистрируются, добавляют свой сервер — система автоматически пингует его, собирает данные (онлайн, версия, MOTD, иконка) и отображает в каталоге. Игроки ищут серверы, голосуют, оставляют отзывы.

---

## 2. Стек технологий

| Компонент | Технология |
|-----------|-----------|
| Backend | PHP 8.1+ |
| База данных | MySQL 8 / MariaDB 10.6+ |
| Frontend | HTML5, CSS3, Vanilla JS |
| Графики | Chart.js 3.x |
| Пинг серверов | PHP (Minecraft Query Protocol / Server List Ping) |
| Автоматизация | Cron (пинг каждые 10 минут) |
| Веб-сервер | Nginx или Apache |
| Кэширование | Файловый кэш (JSON) для главной и каталога |

## 4. Страницы и функционал (публичная часть)

### 4.1 Главная страница `/`

- Заголовок + строка поиска
- Топ-10 серверов по голосам за месяц
- Топ-10 по онлайну прямо сейчас
- Новые серверы (последние 10)
- Общая статистика (всего серверов, игроков онлайн суммарно, голосов сегодня)
- Фильтры: версия, режим, онлайн сейчас

### 4.2 Каталог `/servers`

- Список серверов с пагинацией (20 на страницу)
- Карточка сервера: иконка, название, IP, онлайн, версия, режим, голоса, рейтинг, статус
- Сортировка: по голосам (месяц), по онлайну, по дате, по рейтингу отзывов
- Фильтры: версия, режим, теги, только онлайн
- Поиск по названию и IP

### 4.3 Страница сервера `/server/{id}`

- Полное описание
- Статус (онлайн/оффлайн), текущий онлайн, максимум слотов
- Кнопка "Скопировать IP"
- График онлайна (24ч / 7 дней / 30 дней)
- Кнопка "Голосовать" (если авторизован)
- Счётчик голосов
- Ссылка на сайт сервера
- MOTD сервера
- Версия, режим, теги
- Uptime за 30 дней (% времени онлайн)
- Блок отзывов (список + форма добавления)
- Кнопка "Пожаловаться" на сервер

### 4.4 Регистрация `/register`

- Логин, email, пароль, подтверждение пароля
- Валидация на клиенте и сервере
- Хэширование пароля через password_hash (bcrypt)

### 4.5 Авторизация `/login`

- Логин/email + пароль
- Сессии через PHP sessions
- "Запомнить меня" (cookie token)

### 4.6 Статические страницы `/page/{slug}`

- О проекте, правила, FAQ, контакты
- Контент редактируется из админки
- SEO: title, description

## 5. Личный кабинет `/dashboard`

### 5.1 Главная кабинета `/dashboard`

- Сводка: количество серверов, суммарные голоса, суммарный онлайн
- Список моих серверов (название, статус, онлайн, голоса)
- Уведомления (сервер отклонён, сервер оффлайн давно, и т.д.)

### 5.2 Добавление сервера `/dashboard/add`

- Форма: название, IP, порт, описание, версия, режим, теги, сайт
- Автоматическая проверка доступности при добавлении
- Получение иконки и MOTD автоматически
- Лимит: 5 серверов на аккаунт
- После добавления — статус `pending`, ждёт модерации

### 5.3 Редактирование сервера `/dashboard/edit/{id}`

- Изменение описания, тегов, версии, режима, сайта

- Uptime
- Пиковый онлайн за период

- Смена пароля

- Удаление аккаунта (с подтверждением)

## 6. Админ-панель `/admin`

### 6.1 Дашборд `/admin`

Главная страница админки — сводка всей платформы на одном экране:

- **Статистика за сегодня:**
  - Новых пользователей
  - Новых серверов (добавлено)
  - Голосов за сегодня
  - Отзывов за сегодня
  - Жалоб за сегодня

- **Общая статистика:**
  - Всего пользователей (активных / забаненных)
  - Всего серверов (active / pending / rejected / banned)
  - Всего голосов (за всё время / за месяц)
  - Серверов онлайн прямо сейчас
  - Суммарный онлайн игроков на всех серверах

- **Графики:**
  - Регистрации по дням (за 30 дней)
  - Добавление серверов по дням (за 30 дней)
  - Голоса по дням (за 30 дней)

- **Требует внимания:**
  - Серверов на модерации (pending) — количество + ссылка
  - Новых жалоб (new) — количество + ссылка
  - Серверов оффлайн > 7 дней — количество + ссылка

- **Последние действия:**
  - Лента из admin_log (последние 20 записей)

### 6.2 Управление серверами `/admin/servers`

#### Список серверов
- Таблица: ID, название, IP:порт, владелец, статус, онлайн, голоса, дата добавления
- Фильтры: по статусу (active/pending/rejected/banned), по онлайну, по владельцу
- Поиск: по названию, IP, ID
- Сортировка: по дате, по голосам, по онлайну, по жалобам
- Пагинация (50 на страницу)
- Массовые действия: выбрать несколько → одобрить / отклонить / забанить

#### Модерация новых серверов (pending)
- Отдельная вкладка/фильтр для серверов со статусом `pending`
- Для каждого сервера:
  - Вся информация (название, IP, описание, версия, режим, сайт)
  - Результат автопинга (онлайн ли, сколько игроков, MOTD)
  - Кнопка "Пинг сейчас" — проверить сервер вручную
  - Профиль владельца (когда зарегистрировался, сколько серверов)
  - Кнопки: **Одобрить** / **Отклонить** (с полем для причины)

#### Карточка сервера в админке `/admin/servers/{id}`
- Вся информация о сервере
- История пингов (таблица + график)
- Список голосов (кто, когда)
- Список отзывов
- Список жалоб на этот сервер
- Владелец (ссылка на профиль)
- **Действия:**
  - Изменить статус (active / pending / rejected / banned)
  - Указать причину (при отклонении/бане)
  - Редактировать любое поле сервера
  - Удалить сервер (с подтверждением)
  - Сбросить голоса (месячные или все)
  - Закрепить/открепить в топе (promoted)
  - Принудительный пинг
  - Передать другому владельцу

### 6.3 Управление пользователями `/admin/users`

#### Список пользователей
- Таблица: ID, логин, email, роль, серверов, голосов, дата регистрации, последний вход, статус
- Фильтры: по роли (user/moderator/admin), забаненные, активные
- Поиск: по логину, email, IP
- Сортировка: по дате регистрации, по активности, по количеству серверов
- Пагинация (50 на страницу)

#### Карточка пользователя `/admin/users/{id}`
- Профиль: логин, email, роль, дата регистрации, последний вход, IP
- Его серверы (список со ссылками)
- Его голоса (за какие серверы, когда)
- Его отзывы
- Его жалобы (поданные и полученные)
- История банов
- **Действия:**
  - Изменить роль (user → moderator → admin)
  - Забанить (с причиной, срок или навсегда)
  - Разбанить
  - Сбросить пароль (генерация нового, отправка на email)
  - Удалить пользователя (с подтверждением, каскадно удаляет серверы/голоса/отзывы)
  - Войти как пользователь (для отладки, логируется в admin_log)

### 6.4 Управление жалобами `/admin/reports`

#### Список жалоб
- Таблица: ID, тип (сервер/отзыв/пользователь), объект, от кого, причина, статус, дата
- Фильтры: по статусу (new/in_progress/resolved/rejected), по типу
- Сортировка: по дате (новые первые), по статусу
- Счётчик новых жалоб в меню админки (бейдж)

#### Обработка жалобы `/admin/reports/{id}`
- Информация о жалобе: кто подал, на что, причина, описание
- Объект жалобы (сервер/отзыв/пользователь) — полная информация
- **Действия:**
  - Взять в работу (in_progress)
  - Решить (resolved) — с комментарием что сделано
  - Отклонить (rejected) — с комментарием почему
  - Быстрые действия: забанить сервер, скрыть отзыв, забанить пользователя — прямо из жалобы

### 6.5 Управление отзывами `/admin/reviews`

- Таблица: ID, сервер, автор, оценка, текст (обрезанный), статус, дата
- Фильтры: по статусу (active/hidden/deleted), по оценке
- Поиск: по тексту, по автору, по серверу
- **Действия:**
  - Скрыть отзыв (с причиной)
  - Восстановить скрытый отзыв
  - Удалить отзыв

### 6.6 Управление категориями `/admin/categories`

- Список категорий: название, slug, иконка, порядок, активна ли
- Добавить категорию
- Редактировать категорию
- Изменить порядок (drag-and-drop или стрелки вверх/вниз)
- Деактивировать (не удалять, чтобы не ломать связи)
- Счётчик серверов в каждой категории

### 6.7 Настройки платформы `/admin/settings`

Все настройки хранятся в таблице `settings`, редактируются через форму:

- **Общие:**
  - Название сайта
  - Описание (meta description)
  - Контактный email
  - Ссылки на соцсети (Telegram, VK, Discord)

- **Серверы:**
  - Лимит серверов на аккаунт (по умолчанию 5)
  - Требуется ли модерация новых серверов (да/нет)
  - Максимальная длина описания
  - Разрешённые версии Minecraft (список)
  - Интервал пинга (минуты)
  - Таймаут пинга (секунды)
  - Через сколько неудачных пингов считать сервер оффлайн

- **Голосование:**
  - Кулдаун голосования (часы, по умолчанию 24)
  - Голосование только для авторизованных (да/нет)

- **Отзывы:**
  - Включены ли отзывы (да/нет)
  - Модерация отзывов (да/нет)
  - Минимальная длина отзыва
  - Максимальная длина отзыва

- **Регистрация:**
  - Регистрация открыта (да/нет)
  - Минимальная длина пароля
  - Требуется подтверждение email (да/нет)

### 6.8 Статические страницы `/admin/pages`

- Список страниц: заголовок, slug, опубликована ли, дата изменения
- Создать страницу
- Редактировать страницу (WYSIWYG или HTML-редактор)
- Опубликовать / снять с публикации
- Удалить страницу
- Предпросмотр

### 6.9 Лог действий `/admin/log`

- Фильтры: по администратору, по типу действия, по дате
- Поиск по деталям
- Экспорт в CSV
- Записи не удаляются (только просмотр)

**Что логируется:**
- Одобрение/отклонение/бан сервера
- Бан/разбан пользователя
- Изменение роли пользователя
- Удаление сервера/пользователя/отзыва
- Обработка жалобы
- Изменение настроек платформы
- Редактирование статических страниц
- Вход в аккаунт пользователя
- Сброс голосов
- Изменение категорий

## 7. Система пинга серверов

### 7.1 Механизм

Cron-задача запускает PHP-скрипт каждые 10 минут:

1. Выбирает все серверы со статусом `active`
2. Для каждого сервера отправляет Server List Ping (протокол MC)
3. Получает: онлайн, максимум, MOTD, версию, иконку
4. Записывает результат в `server_stats`
5. Обновляет поля `is_online`, `players_online`, `players_max`, `motd`, `last_ping` в `servers`
6. При неудаче — увеличивает `consecutive_fails`
7. Если `consecutive_fails` > 3 — помечает сервер как оффлайн

### 7.2 Протокол пинга

Minecraft Server List Ping (SLP) — TCP соединение на порт сервера:
- Отправка Handshake пакета
- Отправка Status Request
- Получение JSON ответа с данными сервера
- Таймаут: 5 секунд

### 7.3 Хранение истории

- Записи в `server_stats` хранятся 30 дней
- Старые записи удаляются cron-задачей раз в сутки
- Для графика: за 24ч — все точки, за 7 дней — усреднение по часу, за 30 дней — усреднение по 3 часа

## 8. Система голосования

- Авторизованный пользователь может голосовать 1 раз в 24 часа за каждый сервер
- Проверка по user_id + дата (не по IP, чтобы не блокировать NAT)
- После голоса: votes_total +1, votes_month +1
- votes_month сбрасывается 1-го числа каждого месяца (cron)
- Рейтинг строится по votes_month (голоса за текущий месяц)
- Забаненные пользователи не могут голосовать

---

## 9. Система отзывов

- Авторизованный пользователь может оставить 1 отзыв на каждый сервер
- Оценка от 1 до 5 звёзд + текст
- Средний рейтинг пересчитывается при добавлении/удалении отзыва
- Владелец сервера не может оставить отзыв на свой сервер
- Модератор/админ может скрыть отзыв с указанием причины
- Пользователь может пожаловаться на отзыв

---

## 10. Система жалоб

- Любой авторизованный пользователь может пожаловаться на сервер, отзыв или пользователя
- Категории причин:
  - Сервер: неактуальная информация, мошенничество, накрутка, нарушение правил
  - Отзыв: спам, оскорбления, ложная информация, накрутка
  - Пользователь: мультиаккаунт, накрутка голосов, спам
- Жалоба попадает в очередь модерации
- Модератор/админ обрабатывает жалобу и принимает решение
- Все действия логируются

## 13. Безопасность

- Пароли: bcrypt через password_hash()
- SQL: PDO с prepared statements (никаких конкатенаций)
- XSS: htmlspecialchars() на весь вывод
- CSRF: токены в формах
- Rate limiting: ограничение запросов на голосование, регистрацию, жалобы
- Валидация IP серверов (формат, запрет localhost/приватных сетей)
- Загрузка файлов: проверка MIME-типа, размера, расширения
- Админка: проверка роли на каждой странице, логирование всех действий
- Сессии: привязка к IP + User-Agent, регенерация ID при логине
- Защита от брутфорса: блокировка после 5 неудачных попыток входа (на 15 минут)
